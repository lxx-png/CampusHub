# 👥 团队章程

## 📌 一、团队目标

本团队围绕 CampusHub 项目，采用 AI 协同开发模式，在 10 周内完成核心功能（P0）并实现部分扩展功能（P1/P2），目标包括：

- 构建可运行的校园互助平台系统
- 掌握规范的软件工程流程（Git + PR + CI）
- 探索 AI 在开发、测试、设计中的辅助作用

------

## 🧩 二、角色职责说明

### 需求负责人

负责需求获取与整理，编写需求文档（SRS），进行用例建模，并明确功能范围与优先级。

### 架构负责人

负责技术栈选型与系统架构设计，制定整体技术方案，并编写 ADR（架构决策记录）。

### 开发负责人

负责核心模块的实现与代码规范制定，确保代码质量与系统功能的正确实现。

### 测试负责人

负责制定测试策略、设计测试用例，并在开发过程中持续进行质量保障与缺陷跟踪，确保系统稳定性与可靠性。

------

## 🤖 五、AI 工具链选型与使用

### 🧠 AI 编程助手

- GitHub Copilot
  - 代码补全与函数生成
  - 提升开发效率

👉 选择理由：
GitHub Copilot 与 VS Code 等开发环境深度集成，能够在编码过程中提供实时的代码补全与函数生成建议，从而减少重复性编码工作并提升开发效率，同时有助于在团队开发中保持代码风格的一致性。

------

### 💬 AI 对话工具

- ChatGPT
  - 技术问题解答
  - 架构设计辅助
  - 文档生成
- DeepSeek
  - 代码逻辑分析
  - 算法优化

👉 选择理由：
ChatGPT 适用于技术问题解答、方案设计以及文档生成，能够在需求分析和开发过程中提供高质量的解释与建议，从而提升团队整体决策效率；DeepSeek 在代码理解与逻辑推理方面表现突出，适合复杂代码分析与优化场景，两者结合使用能够形成互补，提高 AI 辅助开发的整体效果。

------

### 🔍 AI 代码审查工具

- CodeRabbit
  - 自动分析代码质量
  - 提供优化建议与风险提示
  - 支持 Pull Request 场景的智能审查

👉 选择理由：
CodeRabbit 作为专门的 AI 代码审查工具，能够基于代码变更自动分析潜在问题并提供改进建议，相比传统人工评审具有更高的效率与覆盖能力，有助于在开发过程中提前发现缺陷并提升代码质量。

------

### 🧪 AI 测试工具

- Testim
  - 自动生成测试流程
  - 自适应 UI 变化（self-healing）
  - 降低测试维护成本

👉 选择理由：
Testim 作为 AI 测试工具，能够通过机器学习自动生成测试流程并在界面发生变化时进行自适应调整，从而减少人工编写和维护测试脚本的成本；相比传统测试方式，其更适合快速迭代开发场景，有助于提升测试效率与系统稳定性。

------

## 🔄 六、开发流程规范

```text
dev → feature → commit → push → PR → merge → dev
```

流程说明：

1. 从 `dev` 创建 `feature` 分支
2. 完成功能开发并提交代码
3. 提交 Pull Request
4. 合并到 `dev` 分支
