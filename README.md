# 📚 CampusHub – 校园互助平台

## 📌 一、项目简介

CampusHub 是一个面向高校学生的互助服务平台，旨在解决校园内信息分散、匹配效率低以及信任机制缺失的问题。

平台支持学生发布和接取互助需求，例如快递代取、学习辅导、二手交易和活动组队等，通过统一的平台实现需求匹配、订单管理与信用评价，从而提升校园协作效率。

------

## 🧩 二、核心功能

- 用户管理：注册、登录、个人信息维护
- 需求发布与浏览：支持分类、筛选与详情查看
- 订单流程：接单 → 确认 → 进行中 → 完成 → 评价
- 信用评价体系：双向评分与信用分机制
- 消息通知：订单状态与互动提醒

------

## 📖 四、技术栈说明

本项目采用前后端分离架构，以提升系统的可维护性与扩展性。后端基于 FastAPI 构建 RESTful API，具备高性能与良好的异步支持，结合 SQLAlchemy 实现数据库操作与模型管理；前端使用 Vue 3 搭配 Vite 构建现代化单页应用，提高开发效率与用户体验。

在数据存储方面，MySQL 用于核心业务数据的持久化管理，Redis 用于缓存与会话管理，以提升系统响应速度。项目通过 GitHub Actions 实现持续集成，确保代码提交后的自动化检查与测试，同时使用 Docker 与 Nginx 进行部署，保证环境一致性与服务稳定性。

此外，引入 GitHub Copilot、ChatGPT 与 DeepSeek 等 AI 工具辅助开发，在代码生成、问题分析与文档编写等方面提升整体开发效率。

## 🤝 八、团队章程（Team Charter）

### 角色职责说明

#### 需求负责人

负责需求获取与整理，编写需求文档（SRS），进行用例建模，并明确功能范围与优先级。

#### 架构负责人

负责技术栈选型与系统架构设计，制定整体技术方案，并编写 ADR（架构决策记录）。

#### 开发负责人

负责核心模块的实现与代码规范制定，确保代码质量与系统功能的正确实现。

#### 测试负责人

负责制定测试策略、设计测试用例，并在开发过程中持续进行质量保障与缺陷跟踪，确保系统稳定性与可靠性。
