# 1. 引言

## 1.1 目的

本文档描述"校园互助服务平台"（CampusHub）的核心需求。该平台面向大学生，支持发布和接单各类校园互助服务（快递代取、学习辅导、二手交易、组队匹配等）。

## 1.2 范围

- **平台类型**：Web + 移动端应用
- **目标用户**：在校大学生（需实名认证）
- **核心业务**：信息发布、任务匹配、订单管理、评价反馈
- **新增业务范围**：
  - 失物招领（问卷高频需求，超过60%受访者提及）
  - 信息咨询与答疑（打破信息差，课程/专业咨询）
  - 社交搭子/组队匹配（运动搭子、课程组队、活动伙伴）
  - 论坛社区（匿名树洞、吐槽、经验分享）

## 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 发布者 | 发布互助需求的用户 |
| 接单者 | 接受任务的用户 |
| 信用积分 | 用户信誉度指标（初始100分） |
| 实名认证 | 通过学号和学校邮箱验证身份 |
| **跑腿** | **代办类任务,如快递代取、外卖配送、物品搬运等** |
| **找搭子** | **寻找志同道合的伙伴,如运动搭子、学习搭子、旅游伙伴等** |
| **情报交换** | **信息咨询、经验分享、打破信息差的服务** |

# 2. 总体描述

## 2.1 产品背景

解决大学生在校园生活中的实际问题：
- 快递取件耗时耗力
- 学习难点需要即时帮助
- 闲置物品缺乏流通渠道
- 活动组队困难

平台通过连接供需双方，构建安全便捷的校园互助生态。

## 2.2 核心功能模块

1. **用户管理**：注册登录、实名认证、个人资料、信用积分
2. **需求发布**：创建任务、设置报酬、上传图片、状态管理
3. **任务浏览**：分类筛选、搜索、详情查看、在线接单
4. **交流沟通**：站内消息、任务评论
5. **订单管理**：我的发布/接单、状态跟踪、确认完成
6. **评价反馈**：双向评价、星级评分、举报投诉
7. **智能推荐**：基于兴趣和位置的个性化推荐
8. **后台管理**：用户管理、内容审核、数据统计
9. **失物招领**：**专门的失物发布与认领模块（问卷超60%提及）**
10. **论坛社区**：**匿名树洞、经验分享、话题讨论（约30%用户期望）**
11. **资讯问答**：**课程咨询、专业建议、信息共享（约40%用户重视）**

## 2.3 用户特征

| 用户类型 | 特征 | 使用频率 |
|----------|------|----------|
| 普通学生 | 18-25岁，有互助需求 | 每周2-5次 |
| 高频用户 | 经常发布或接单 | 每日多次 |
| 管理员 | 平台运营人员 | 工作日全天 |
| **志愿管理员** | **热心学生志愿者，协助内容审核和社区维护（约40%表示愿意参与）** | **不定期** |

## 2.4 约束条件

### 技术约束
- **前端**：React/Vue (Web) + Flutter/React Native (移动端)
- **后端**：Java/Python/Node.js + MySQL + Redis
- **部署**：Linux + 云平台（阿里云/腾讯云）

### 业务约束
- 仅限在校学生，需实名认证
- 禁止违法、作弊类任务
- 单次报酬上限500元，任务有效期最长7天
- **隐私保护**：**学号认证需脱敏处理，避免个人信息泄露（约15%用户担忧）**
- **匿名功能**：**部分场景（如树洞、出物）需提供匿名选项**

### 法规约束
- 符合《网络安全法》实名制要求
- 遵守《个人信息保护法》
- 遵循《电子商务法》

### **社区管理约束（基于问卷反馈）**
- **内容审核**：**发帖前审查、敏感词过滤、举报机制**
- **言论规范**：**禁止不良言论、攻击性语言、虚假信息**
- **时效性管理**：**过期内容自动清理、帖子置顶/下沉机制**
- **隐私保护**：**做好防黑客措施、每人同时接受委托有限制**

## 2.5 假设条件

- 用户具备基本网络应用能力
- 学校网络基础设施完善
- 用户身份信息真实有效
- 移动支付普及率高
- **学校官方支持**：**希望能得到学校官方协助进行身份认证（约20%用户提及）**

# 3. 具体需求

## 3.1 外部接口需求

### 3.1.1 用户界面

**设计风格**
- 扁平化设计，主色调：校园蓝（#1E88E5）
- 响应式布局，适配手机、平板、PC

**关键页面**
- **首页**：搜索框、分类入口（**增加失物招领醒目入口**）、任务列表
- **发布页**：类型选择（**增加失物招领、找搭子、情报交换**）、信息填写、图片上传
- **详情页**：任务信息、发布者资料、操作按钮
- **个人中心**：我的任务、信用积分、设置
- **论坛页面**：**匿名树洞、话题讨论、点赞关注**
- **失物招领专区**：**分类展示（遗失/拾得）、快速匹配、认领流程**

**交互体验（基于问卷建议）**
- **页面简洁**：**广告少、易交互、流畅不卡顿（约25%用户强调）**
- **多种标签**：**给帖子增加多种标签便于搜索**
- **固定帖子时长**：**重要帖子可置顶**
- **关注/@功能**：**可增加关注、@功能（约10%用户建议）**

### 3.1.2 第三方接口

- **短信服务**：验证码发送（阿里云/腾讯云）
- **推送服务**：消息提醒（极光推送/个推）
- **地图定位**：位置获取（高德/腾讯地图）
- **云存储**：图片文件存储（七牛云/阿里云OSS）

### 3.1.3 API规范

- **协议**：HTTPS + RESTful API
- **数据格式**：JSON
- **统一响应**：`{code, message, data}`

## 3.2 功能需求

### FR-01：用户注册与登录 ✏️

**描述**：新用户通过手机号/学号注册，已注册用户登录  
**优先级**：高

**输入**：账号、密码、验证码  
**处理**：
1. 验证账号密码格式
2. 检查账号状态
3. 密码加密验证
4. 生成Token

**输出**：登录成功返回Token和用户信息  
**规则**：
- 密码6-20位，含字母和数字
- 连续输错5次锁定30分钟
- Token有效期7天
- **认证方式**：**支持学号认证，但需脱敏处理；也可通过校内邮箱辅助认证（约15%用户建议交叉验证）**

---

### FR-02：实名认证 ✏️

**描述**：通过学号和学校邮箱验证身份  
**优先级**：高

**输入**：学号、姓名、学校邮箱  
**处理**：
1. 验证学号格式
2. 发送验证邮件
3. 更新认证状态

**输出**：认证成功获得标识  
**规则**：未认证用户只能浏览，不能发布/接单
- **隐私保护**：**学号在显示时部分隐藏，防止隐私泄露**

---

### FR-03：个人资料管理 ✏️

**描述**：编辑个人信息  
**优先级**：高

**输入**：昵称、头像、性别、学院、专业、简介  
**处理**：验证并保存  
**输出**：保存结果  
**规则**：昵称不含敏感词，7天可修改一次
- **新增 匿名设置**：**用户可设置是否匿名显示（针对特定场景如树洞、出物）**

---

### FR-04：信用积分管理 ✏️

**描述**：系统自动计算用户信用分  
**优先级**：高

**初始积分**：100分  
**加分规则**：
- 完成订单+2分
- 获得好评+3分
- 连续活跃+5分

**减分规则**：
- 取消订单-5分
- 获得差评-10分
- 被举报核实-20分

**规则**：
- 低于60分限制发布
- 低于40分暂停使用
- 每日上限增加20分
- **新增 奖惩机制**：**高信用分用户享有更大自由度（如同时接更多单）**

---

### FR-05：发布互助任务 ✏️

**描述**：创建并发布互助需求  
**优先级**：高

**输入**：
- 任务类型（快递/学习/二手/组队/其他/**失物招领/找搭子/情报交换/论坛发帖**）
- 标题（≤50字）、描述（≤500字）
- 报酬（0-500元）、截止时间（≤7天）
- 图片（最多9张）、标签（最多5个）

**处理**：
1. 验证认证状态和输入
2. **新增 内容审核**：**发帖前审查、敏感词过滤（约30%用户强调）**
3. 创建任务记录
4. 推送给相关用户

**输出**：发布成功跳转详情页  
**规则**：每人每天最多发布10条
- **新增 帖子类型**：**支持固定帖、精华帖、置顶帖**

---

### FR-06：任务浏览与搜索 ✏️

**描述**：查看和搜索任务列表  
**优先级**：高

**输入**：筛选条件、关键词、排序方式  
**处理**：
1. 查询数据库
2. 应用筛选和排序
3. 分页返回

**输出**：任务列表（标题、报酬、时间等）  
**性能**：响应时间<500ms
- **新增 智能推荐**：**根据用户年级、专业提供定制化服务**
- **新增 分类检索**：**优化推荐算法，定时清理过期内容**

### FR-07：任务详情查看 ✅

**描述**：查看任务完整信息  
**优先级**：高

**输入**：任务ID  
**处理**：获取任务、发布者、评论信息  
**输出**：任务详情页面

### FR-08：在线接单 ✏️

**描述**：申请接受任务  
**优先级**：高

**输入**：任务ID、留言（可选）  
**处理**：
1. 验证认证和信用分（≥80）
2. 检查任务状态和并发数（≤5）
3. 创建接单记录
4. 通知发布者

**输出**：接单结果  
**规则**：每人同时最多接5个任务

### FR-09：接单确认 ✅

**描述**：发布者选择合适的接单者  
**优先级**：高

**输入**：任务ID、接单者ID  
**处理**：
1. 确认接单者

**输出**：确认结果

### FR-10：站内消息 ✏️

**描述**：用户间即时通讯  
**优先级**：高

**输入**：接收者ID、消息内容  
**处理**：
1. 内容安全检测
2. 保存消息
3. WebSocket推送

**输出**：发送结果

### FR-11：订单管理 ✏️

**描述**：查看和管理我的任务  
**优先级**：高

已取消   重新发布   已超时

# 6. 附录

## 4.1 核心数据表

### 用户表 (users)
- 基本信息：ID、用户名、密码、手机号、学号
- 个人资料：昵称、头像、性别、学院、专业
- 状态信息：信用积分、认证状态、账号状态
- **新增**：**匿名设置、关注数、粉丝数**

### 任务表 (tasks)
- 任务信息：ID、发布者ID、标题、描述、类型
- 附加信息：报酬、截止时间、图片、标签
- 状态信息：状态、浏览量、申请数
- **新增**：**是否匿名、帖子类型（普通/置顶/精华）、过期时间**

### 订单表 (orders)
- 订单信息：ID、任务ID、发布者ID、接单者ID
- 状态信息：订单状态、约定价格、完成时间
- **新增**：**完成凭证、最晚完成时间、自动确认标志**

### 评价表 (reviews)
- 评价信息：ID、订单ID、评价者、被评价者
- 评价内容：评分、内容、标签、是否匿名

### 消息表 (messages)
- 消息信息：ID、发送者、接收者、关联任务
- 消息内容：内容、类型、是否已读

### **失物表 (lost_found)**
- **基本信息：ID、发布者ID、类型（遗失/拾得）**
- **物品信息：名称、分类、时间、地点、描述、照片**
- **状态：是否找到、是否认领、认领验证状态**

### **论坛帖子表 (forum_posts)**
- **基本信息：ID、作者ID、标题、内容**
- **帖子属性：类型（树洞/讨论/分享）、是否匿名、点赞数、评论数**
- **状态：是否置顶、是否精华、是否审核通过**

### **问答表 (qa_posts)**
- **问题信息：ID、提问者ID、标题、内容、悬赏积分**
- **回答信息：回答者ID、内容、是否最佳答案、点赞数**

## 4.2 业务流程

**发布流程**：
```
登录 → 认证 → 填写信息 → 审核 → 发布成功
```

**接单流程**：
```
浏览 → 详情 → 申请 → 确认 → 执行 → 完成 → 评价
```

**失物招领流程**：
```
发布失物/拾得信息 → 智能匹配 → 申请认领 → 验证身份 → 确认归还 → 双方评价
```

**论坛发帖流程**：
```
选择板块 → 编写内容（可匿名）→ 内容审核 → 发布成功 → 互动（点赞/评论/转发）
```

**问答流程**：
```
提出问题（设置悬赏）→ 多人回答 → 标记最佳答案 → 分配积分 → 双方评价
```
