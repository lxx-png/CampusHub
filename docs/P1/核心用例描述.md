# 核心用例描述

## 用例一：发布需求（UC4）

**用例名称**： 发布互助需求
**参与者：** 👤需求方

**前置条件**： 用户已完成注册并成功登录系统，当前角色为需求方，且已完善基本个人资料。

**基本流：**
需求方进入“发布需求”页面后，系统展示需求填写表单。用户填写需求标题、描述、分类、地点、时间以及报酬或交换条件等信息，确认无误后点击“发布”按钮。系统首先对输入内容进行合法性校验，包括必填项检查和格式检查；校验通过后，将需求信息保存至数据库，并自动将需求状态设置为“待接单”。随后系统向需求方发送发布成功通知，并跳转至该需求的详情页面。

**替代流：**
若用户填写信息不完整，例如标题、分类或时间为空，系统应提示“必填项不能为空”，并停留在当前页面等待用户补充；若系统在保存过程中出现异常，例如数据库写入失败，则提示“发布失败，请稍后重试”；若需求内容中包含违规信息或敏感词，则系统拒绝发布并向用户说明具体原因。

**后置条件：**
新需求被成功创建，状态为“待接单”，可在需求列表中展示，并可被服务方浏览和接单。

## 用例二：接单（UC8）

**用例名称：** 接单服务
**参与者：** 👤服务方

**前置条件：** 用户已成功登录系统，当前角色为服务方，且所选需求状态为“待接单”。

**基本流：**
服务方首先浏览需求列表，选择感兴趣的任务进入需求详情页面。确认任务内容后点击“接单”按钮，系统对当前需求状态进行校验，确认其仍处于“待接单”状态。校验通过后，系统更新订单状态为“已接单”，同时生成对应订单记录，并向需求方发送接单通知。随后系统跳转至订单流程页面，供双方继续后续操作。

**替代流：**
若该需求已被其他服务方抢先接单，系统应提示“该需求已被接单”；若当前用户未登录，则自动跳转至登录页面；若需求方尝试执行接单操作，系统应提示“当前角色无权限”。

**后置条件：**
订单记录成功生成，订单状态更新为“已接单”，双方进入后续服务流程。

## 用例三：确认完成（UC11）

**用例名称：** 确认订单完成
**参与者：** 👤需求方、👤服务方

**前置条件：** 当前订单状态为“进行中”，且双方已完成线下服务。

**基本流：**
服务方完成任务后点击“申请完成”，系统将订单状态更新为“待确认完成”，并向需求方发送确认通知。需求方查看服务结果后点击“确认完成”，系统随即将订单状态更新为“已完成”，同时开放双向评价入口，并根据评价结果更新双方信用分。

**替代流：**
若需求方认为任务尚未完成或服务结果不符合要求，可拒绝确认，系统将订单状态恢复为“进行中”；若在规定时间内需求方未进行确认，系统应自动发送提醒通知，必要时可触发平台超时处理机制。

**后置条件：**
订单流程结束，状态变为“已完成”，双方可进行评价，系统同步更新信用分及信用等级。
