# 1. 引言（Introduction）

## 1.1 目的（Purpose）
本文档定义“校园互助服务平台”的软件需求，明确系统功能、性能、约束等，为开发人员、测试人员及相关利益相关者提供统一参考。

## 1.2 范围（Scope）
本系统是一个面向大学生的在线平台，支持用户发布和接受互助任务，包括但不限于：

- 快递代取
- 学习辅导
- 二手交易
- 组队匹配

系统支持用户注册登录、任务管理、评价体系等功能，旨在提升校园资源利用效率与学生之间的互助协作。

# 2. 总体描述（Overall Description）

## 2.1 产品视角（Product Perspective）
系统采用前后端分离架构：

- 前端：Web/移动端（React + TailwindCSS）
- 后端：RESTful API 服务
- 数据库：MySQL / MongoDB

## 2.2 产品功能（Product Functions）
系统主要功能包括：

- 用户管理（注册、登录、认证）
- 任务发布与管理
- 接单与订单管理
- 分类浏览与搜索
- 用户评价与信用体系
- 消息通知

## 2.3 用户特征（User Characteristics）

| 用户类型 | 特征 |
|----------|------|
| 普通学生 | 熟悉手机/网页操作 |
| 发布者 | 有需求发布能力 |
| 接单者 | 有时间完成任务 |

## 2.4 约束（Constraints）
- 必须支持校园身份认证（如学号/邮箱）
- 系统需符合数据隐私保护规范
- 需兼容主流浏览器

## 2.5 假设与依赖（Assumptions and Dependencies）
- 用户具备基本互联网使用能力
- 校园网络环境稳定
- 第三方服务（短信/支付）可用

## 3.3 非功能需求（Non-Functional Requirements）

### 3.3.1 性能需求（Performance）
- 响应时间 < 2 秒
- 支持 ≥ 1000 并发用户

### 3.3.2 安全性（Security）
- 数据加密传输（HTTPS）
- 用户密码加密存储
- 防止SQL注入

### 3.3.3 可用性（Usability）
- UI简洁易用
- 学习成本低

### 3.3.4 可靠性（Reliability）
- 系统可用性 ≥ 99%
- 数据备份机制

### 3.3.5 可维护性（Maintainability）
- 模块化设计
- 清晰代码结构

### 3.3.6 可扩展性（Scalability）
- 支持未来增加新功能（如跑腿、兼职）
